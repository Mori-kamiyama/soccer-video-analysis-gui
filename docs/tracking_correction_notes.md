# 選手トラッキング補正メモ

## 目的

SAM/YOLOで得た検出・追跡結果を、人間が短時間で直せる形にする。
完全自動のID維持より、交差・欠落・入れ替わりをすばやく発見して局所補正できる状態を優先する。

## 現在の方針

- `player_tracker_gui.py` を手動補正の中心にする。
- 左クリックで基準トラックを選び、別トラックを右クリックすると、このフレーム以降の担当選手をスワップする。
- 同じ選手が同時に複数表示されているフレーム、未割当IDがあるフレームへジャンプできるようにする。
- 断片IDの統合は `player_merge_gui.py`、選手名簿への割当と入れ替わり補正は `player_tracker_gui.py` に分ける。

## 仮説

- サッカー映像の交差では、検出精度よりもReIDまたは手動補正UIの不足でIDが崩れている可能性が高い。
- SAM2はマスクの継続には強いが、全選手を長時間ほぼ無操作で識別し続ける用途では、YOLO + tracker + 補正GUIの方が運用しやすい。
- 交差時の入れ替わりは「そのフレーム以降だけスワップ」できれば、修正コストを大きく下げられる。

## 次に試す候補

- UltralyticsのBoT-SORTでReIDを有効化し、ByteTrackとのID維持を比較する。
- チーム分類を先に安定化し、別チーム同士の誤接続をトラッカー側で抑える。
- 交差・消失フレームだけを抽出して、補正GUIで重点的に確認する。

## 方針変更: 個人IDを追わない補正

交差でIDが重なる・消える・入れ替わる問題は、個人識別を維持しようとすると修正コストが高い。
分析の第一段階では、各フレームごとに「選手ボックス」「足元位置」「チーム色」だけを正確にする方針に寄せる。

- `photo_annotator.py` はチーム色 A/B/? の補正GUIとして使う。
- `box_corrections.csv` は引き続き非破壊の編集ログとして使う。
- `build_corrected_detections.py` で補正済みの `outputs/detections_corrected.csv` を作る。
- その後 `step5_transform_to_pitch.py` に渡せば、個人IDなしでピッチ座標化できる。

## ボックス修正GUIの追加チェック

- 1フレームのボックス数が22人でない場合、23人以上の場合、A/B が同数でない場合、不明色が残っている場合は赤い警告を出す。
- `J` キーまたは「次の警告」ボタンで、次の警告フレームへ移動する。
- 追加したボックスや不明色のボックスは、胴体帯の緑比率から自動で A/B 判定する。
- 自動判定はあくまで初期補助なので、誤判定は `A`/`B`/`U` キーで色を選んでクリック修正する。
- 100件程度の手修正を保存したら、「注釈から色判定を学習」で `outputs/team_color_calibration.json` に判定境界を保存する。
- 保存済み注釈では B 側の方が緑比率が高く、A/B 表示が逆の可能性がある。目視で逆なら「全F 緑/黒入替」を使う。
- 全F入替後は全ボックスが色修正済み扱いになり、手注釈学習に余計な検出が混ざる。先頭50フレームなど範囲を絞る場合は `train_team_color_model.py --frame-start 0 --frame-end 50 --mode cluster` で教師なしクラスタ学習する。
- 2026-06-23時点では先頭50フレームから `outputs/team_color_knn_model.json` を作成済み。結果は samples=630, A=393, B=237, loo_acc=0.970。
